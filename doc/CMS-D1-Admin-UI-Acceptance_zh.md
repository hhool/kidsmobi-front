# CMS D1 管理后台 UI 人工点击验收记录（中文）

时间：2026-06-30 16:08 CST

验收页面：

1. https://kidsmobi.pages.dev/#cms

验收方式：

1. 通过“开发者一键快捷登录”进入 Admin 控制台。
2. 逐个点击左侧六个业务模块。
3. 在产品中心执行一次“新增 -> 保存”触发写入前置校验。

部署说明：

1. 验收时请将 `VITE_CMS_API_BASE_URL` 固定到已部署的 Worker CMS API。
2. 若需要后端资源检查，请将 `VITE_SCRAPE_API_BASE_URL` 或 `SCRAPE_KIDSMOBILE_API_BASE_URL` 固定到已部署的 Worker API。

## 1. 模块可达性

结果：六模块均可进入，标题与主操作按钮均出现。

1. 产品中心：显示“产品中心”标题；可见“初始化产品/导入 JSON/导出 JSON/新增产品”。
2. 品类管理：显示“品类管理”标题；可见“初始化品类/新增品类”。
3. 场景管理：显示“场景管理”标题；可见“新增场景”。
4. 评测中心：显示“评测中心”标题；可见“发布新评测”。
5. 选购指南：显示“选购指南”标题；可见“撰写指南”。
6. 全球资讯：显示“全球资讯”标题；可见“发布资讯”。

结论：导航与页面渲染通过。

## 2. 交互验证

产品中心新增流程：

1. 点击“新增产品”进入编辑态。
2. 点击“保存并发布”时弹窗提示：`Please enter product name in both languages.`

结论：表单前置校验生效（未填关键字段时阻断提交）。

## 3. 运行异常与观察

在开发者快捷登录态下观察到：

1. 控制台多次出现 Firestore 报错：`Missing or insufficient permissions`（`operationType=list`, `path=scenarios`）。
2. 顶部状态徽标显示：`D1: 未就绪`。
3. 页面提示当前为开发者影子账户，发布/强制同步会被安全规则拒绝。

解读：

1. 当前“开发免登”仅用于 UI 预览，不等价于真实管理员鉴权。
2. 若需验证真实持久化写入链路，需改用 Google 管理员账号登录或在部署环境固定 `VITE_CMS_API_BASE_URL` 并走 Worker CMS API。

## 4. 验收结论

阶段结论：

1. UI 可达性：通过。
2. 模块导航：通过。
3. 表单前置校验：通过。
4. 开发免登态下真实写入：不通过（符合预期限制，非功能回归缺陷）。

## 5. 后续建议

1. 使用真实管理员账号进行二次人工验收（重点：新增/编辑/删除成功提示与列表刷新）。
2. 在前端部署平台固化对应的 API 基址变量，不要依赖示例域名。
3. 验收完成后，补充一版“真实管理员态 UI 验收”文档作为发布门禁。
